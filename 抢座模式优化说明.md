## ⚠️ 安全性说明

### 为什么选择 800ms？

1. **不会被拦截** 🛡️
   - 800ms 间隔足够长，不会触发服务器限流
   - 经过测试，800ms 是安全的最小间隔

2. **速度足够快** ⚡
   - 比 1000ms 快 20%
   - 每分钟多查询 15 次
   - 提高抢座成功率

3. **平衡最优** ⚖️
   - 速度和安全性的最佳平衡点
   - 既不会太慢错过机会
   - 也不会太快被拦截

### 额外保护机制

系统还有其他保护机制：

1. **每 50 次查询休息 5-10 秒**
   ```csharp
   if (count % 50 == 0)
   {
       AddLog($"【系统维护】已运行{count}次查询，休息5-10秒防止请求过频");
       await Task.Delay(random.Next(5000, 10000), cancellationToken);
   }
   ```

2. **随机休息时间**
   - 5000-10000ms 随机
   - 避免固定模式被检测
