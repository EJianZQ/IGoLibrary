# 🚀 IGoLibrary 一键启动使用指南

## 快速开始

### 方法 1: 双击启动（推荐）

1. 找到项目根目录下的 `启动应用.sh` 文件
2. 双击运行（如果无法双击，请看下方"首次使用设置"）
3. 应用会自动编译并启动

### 方法 2: 终端启动

```bash
cd /Users/apple/PycharmProjects/IGoLibrary
./启动应用.sh
```

## 首次使用设置

### macOS 双击运行设置

如果双击 `.sh` 文件无法运行，需要进行以下设置：

#### 步骤 1: 设置默认打开方式

1. 右键点击 `启动应用.sh` 文件
2. 选择"显示简介"（或按 `⌘ + I`）
3. 在"打开方式"部分，选择"终端.app"（Terminal.app）
4. 点击"全部更改..."按钮

#### 步骤 2: 赋予执行权限（如果还没有）

打开终端，执行：
```bash
chmod +x /Users/apple/PycharmProjects/IGoLibrary/启动应用.sh
```

#### 步骤 3: 允许运行未签名的脚本

如果 macOS 提示"无法打开，因为它来自身份不明的开发者"：

1. 打开"系统偏好设置" → "安全性与隐私"
2. 在"通用"标签页，点击"仍要打开"按钮
3. 或者在终端执行：
```bash
xattr -d com.apple.quarantine /Users/apple/PycharmProjects/IGoLibrary/启动应用.sh
```

## 常见问题

### Q1: 双击后没有反应？

**A**: 请检查以下几点：
1. 是否已设置默认打开方式为"终端.app"
2. 是否已赋予执行权限（`chmod +x`）
3. 是否被 macOS 安全机制阻止（查看"安全性与隐私"设置）

### Q2: 提示"找不到 dotnet 命令"？

**A**: 需要安装 .NET SDK：
```bash
# 使用 Homebrew 安装
brew install --cask dotnet-sdk

# 或者从官网下载安装
# https://dotnet.microsoft.com/download
```

### Q3: 提示"找不到项目目录"？

**A**: 请确保：
1. 脚本文件在项目根目录下
2. 项目根目录下有 `IGoLibrary.Mac` 文件夹
3. 没有移动或重命名项目文件夹

### Q4: 如何切换模拟模式？

**A**: 编辑 `IGoLibrary.Mac/App.axaml.cs` 文件：
```csharp
// 启用模拟模式
private const bool IsSimulationMode = true;

// 禁用模拟模式（使用真实服务）
private const bool IsSimulationMode = false;
```

### Q5: 如何停止应用？

**A**:
- 在应用窗口中点击关闭按钮
- 或在终端窗口按 `Ctrl + C`

## 高级用法

### 自定义启动参数

如果需要传递启动参数，可以修改脚本最后的 `dotnet run` 命令：

```bash
# 传递参数示例
dotnet run -- --arg1 value1 --arg2 value2
```

### 后台运行

如果需要后台运行（不阻塞终端）：

```bash
nohup ./启动应用.sh > app.log 2>&1 &
```

### 查看详细日志

脚本会显示所有编译和运行日志，如果需要保存日志：

```bash
./启动应用.sh 2>&1 | tee startup.log
```
