## ✨ 新增功能

### 🔍 生产环境配置验证（关键功能）

启动脚本现在会**自动验证**生产环境配置，确保启动的是真实环境：

1. **检查 Mock 开关**
   - ✅ 验证 `IsSimulationMode = false`
   - ❌ 如果为 `true`，会警告并询问是否继续

2. **检查时间模拟**
   - ✅ 验证 `EnableTimeSimulation = false`
   - ❌ 如果为 `true`，会警告并询问是否继续

3. **自动重新编译**
   - 自动执行 `dotnet clean`
   - 自动执行 `dotnet build --configuration Release`
   - 确保配置修改生效

## 🔍 验证项目说明

**生产环境要求**: `false`

**如果为 `true`**:
- ❌ 使用模拟服务（MockGetLibInfoService、MockPrereserveSeatService）
- ❌ 无法连接真实服务器
- ❌ 无法进行真实抢座

**生产环境要求**: `false`

**如果为 `true`**:
- ❌ 时间固定在 19:59:45
- ❌ 无法使用真实北京时间
- ❌ 无法正常抢座

## ⚠️ 常见问题

### Q1: 脚本提示配置错误，但我已经修改了配置？

**A**: 需要重新运行脚本，脚本会自动重新编译：
```bash
./启动应用.sh
```

脚本会自动执行：
- `dotnet clean` - 清理旧编译文件
- `dotnet build --configuration Release` - 重新编译

---

### Q2: 我想在测试模式下运行，怎么办？

**A**: 当脚本检测到测试模式时，会询问是否继续：
```
是否继续启动？(yes/no): yes
```

输入 `yes` 即可继续以测试模式启动。

---

### Q3: 如何快速切换回测试模式？

**A**: 修改配置文件：
```csharp
// App.axaml.cs
private const bool IsSimulationMode = true;

// GrabSeatViewModel.cs
private const bool EnableTimeSimulation = true;
```

然后重新运行脚本。

---

### Q4: 编译失败怎么办？

**A**: 脚本会自动显示详细的编译错误信息，根据错误提示修复代码后重试。
