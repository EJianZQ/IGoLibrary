# 🎯 战前演习使用指南

## 功能简介

**战前演习**是增强版的系统自检工具，专为每天 19:50 的抢座前检查设计。通过全链路网络测试，确保：
- ✅ 网络是通的
- ✅ 服务器是活的
- ✅ Cookie 是热的🔥

只要这一步通过，你就能确信系统已就绪，可以放心抢座！

---

## 使用方法

### 1. 打开应用

```bash
cd IGoLibrary.Mac
dotnet run
```

### 2. 切换到"明日预约"页面

应用启动后会自动切换到明日预约页面。

### 3. 点击"系统自检"按钮

在明日预约页面，点击右侧的"系统自检"按钮。

### 4. 查看战前演习报告

系统会在右侧日志区显示详细的检测报告。

## 检测项目（共9项）

### 1. 北京时间获取 ⏰
- **检测内容**：验证系统时区设置是否正确
- **通过标准**：成功获取北京时间（UTC+8）
- **失败原因**：时区配置错误

### 2. Cookie 检查 🍪
- **检测内容**：验证 Cookie 是否设置且格式正确
- **通过标准**：Cookie 包含 `Authorization` 和 `SERVERID` 字段
- **失败原因**：未登录或 Cookie 格式错误

### 3. 图书馆绑定 📚
- **检测内容**：验证是否已绑定图书馆
- **通过标准**：已绑定图书馆且有 LibID
- **失败原因**：未在登录页面绑定图书馆

### 4. 查询语法 📝
- **检测内容**：验证 GraphQL 查询语法是否配置
- **通过标准**：查询语法已配置
- **失败原因**：未绑定图书馆导致查询语法未生成

### 5. 全链路网络测试 🌐（核心）

## 使用建议

### 每天 19:50 运行战前演习

建议在每天 **19:50** 运行一次战前演习，确保系统就绪：

1. **19:50** - 运行战前演习
2. **19:55** - 如果发现问题，立即修复
3. **19:59:50** - 系统自动进入准备状态
4. **20:00:00** - 准时开始抢座

### 快速修复指南

#### 问题1：Cookie 失效
**症状**：403 Forbidden 或 401 Unauthorized
**解决**：
1. 切换到登录页面
2. 点击"获取二维码"
3. 使用微信扫码登录
4. 重新运行战前演习

#### 问题2：网络不通
**症状**：DNS 解析失败或请求超时
**解决**：
1. 检查 WiFi/以太网连接
2. 尝试访问其他网站验证网络
3. 更换网络（如切换到手机热点）
4. 重新运行战前演习

#### 问题3：未设置座位
**症状**：预约座位检查未通过
**解决**：
1. 点击"刷新座位"
2. 选择心仪的座位
3. 点击"添加到预约列表"
4. 重新运行战前演习

#### 问题4：未绑定图书馆
**症状**：图书馆绑定检查未通过
**解决**：
1. 在登录页面点击"加载图书馆列表"
2. 选择目标图书馆楼层
3. 点击"绑定图书馆"
4. 重新运行战前演习

## 技术细节

### 全链路网络测试原理

1. **DNS 解析**：将域名 `wechat.v2.traceint.com` 解析为 IP 地址
2. **TCP 连接**：建立到服务器的 TCP 连接
3. **HTTP 请求**：发送 GraphQL 查询请求
4. **数据解析**：解析返回的座位数据
5. **结果验证**：验证座位数量 > 0

### 错误诊断机制

系统会捕获以下异常类型并提供详细诊断：

- `HttpRequestException` - HTTP 请求异常
  - DNS 解析失败
  - 403 Forbidden
  - 401 Unauthorized
  - 500 Internal Server Error
  - 其他 HTTP 错误

- `TaskCanceledException` - 请求超时

- `GetLibInfoException` - 获取图书馆信息异常
  - Cookie 相关问题
  - 数据未找到

- `Exception` - 其他未知异常

### 响应时间测量

```csharp
var startTime = DateTime.Now;
var testLib = await GetLibInfo(...);
var endTime = DateTime.Now;
var responseTime = (endTime - startTime).TotalMilliseconds;
```

## 常见问题

### Q1: 为什么要在 19:50 运行战前演习？

**A**: 19:50 距离抢座时间（20:00）还有10分钟，足够时间发现并修复问题。如果在 19:59 才发现 Cookie 失效，就来不及重新登录了。

### Q2: 全链路网络测试会影响抢座吗？

**A**: 不会。战前演习只是读取座位信息，不会进行预约操作。而且测试完成后会立即释放连接，不会占用资源。

### Q3: 如果战前演习失败了怎么办？

**A**: 根据日志中的诊断结果和解决方案进行修复，然后重新运行战前演习，直到通过为止。

### Q4: 可以在其他时间运行战前演习吗？

**A**: 可以。战前演习可以随时运行，用于检查系统状态。但建议在抢座前（19:50）运行一次，确保万无一失。

### Q5: 网络质量评级对抢座有什么影响？

**A**:
- **优秀（<500ms）**：抢座成功率最高
- **良好（<1s）**：正常抢座，成功率较高
- **一般（<2s）**：可能影响抢座速度，建议优化网络
- **较慢（>2s）**：抢座成功率降低，强烈建议更换网络
